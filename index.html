What is Cat-tastrophe?

Cat-tastrophe is a puzzle game where cats are hidden under a grid of squares. Your job: reveal every safe square without clicking a cat.

🖱️

Left-click → Reveal

Clicking a square opens it. If it's a cat — game over. If it's safe, it shows how many cats are in the 8 surrounding squares (0–8). A 0 means no cats nearby, so the game auto-opens all neighbours too.

Right-click → Collar

Right-clicking plants a collar on a square you think is a cat. Collared squares can't be opened by accident. Right-clicking again removes the collar. The cat counter updates to show remaining cats.

🏆

How you win

Reveal every single safe square. You don't need to collar all the cats — you just need to open every cell that isn't a cat. When the last safe cell opens, you win instantly.

🐱

First click is always safe

Cats are placed after your first click. The code keeps a "safe zone" — the cell you clicked plus its 8 neighbours — and never puts a cat in that zone. So your very first click always opens a nice big area.

🔢

What the numbers mean

A revealed square shows a number 1–8. That number tells you exactly how many of the 8 squares touching it contain cats. Use this to logically deduce which squares are safe to click.

⚡

The flood fill shortcut

When you reveal an empty square (shows nothing, no cats nearby), the game automatically opens all connected empty squares — sometimes revealing a huge area at once. This uses a BFS algorithm.

Under the hood

How the code is structured

The game is split into two classes and 16 functions. Click any highlighted line to see what it does.

📦
class Cell
Data container · 100 instances

A class is a template. Cell is a template for one board square. The game creates 100 copies — one per square. Each copy is called an instance and stores its own data.

class Cell: is_cat = False # cat hiding here? is_revealed = False # has player opened it? is_collared = False # player put a collar here? neighbour_count = 0 # how many cats nearby? button = None # the button widget
💡No logic here — Cell only stores facts. Think of it as a sticky note attached to each square.
🎮
class CatstropheGame
Brain of the game · 1 instance

This class does everything: draws the window, handles clicks, places cats, runs the timer, and decides if you've won or lost. Only one instance is ever created — game = CatstropheGame().

class CatstropheGame: board = [] # 10×10 grid of Cell objects game_over = False # are clicks disabled? first_click= True # has first click happened? start_time = None # when did the game start? # ... + 15 methods
✅All logic lives here. When you click a button, CatstropheGame decides what to do with it.
🔗How they connect: CatstropheGame creates 100 Cell objects and stores them in self.board[row][col]. To check if square (3,4) is a cat: self.board[3][4].is_cat. That's it — read the Cell's variable.

_build_board() — creates the grid
Uses two nested for loops (row 0→9, col 0→9) to create 100 Cell objects. Each Cell gets a Button widget (the clickable square), and its left/right-click handlers are wired up right here — no separate "connect" step needed.
for row in range(10): # 0,1,2...9 for col in range(10): # 0,1,2...9 cell = Cell() # make a new cell cell.button = Button(...) # attach a clickable button btn.configure(command=lambda r=row,c=col: self.left_click(r,c)) btn.bind("<Button-3>", lambda e,r=row,c=col: self.right_click(r,c)) self.board[row][col] = cell
place_cats(first_row, first_col) — called on first click
Randomly places 10 cats, but never in the "safe zone" (first-clicked cell + its 8 neighbours). Uses a while loop that keeps picking random positions until 10 valid ones are found. Boolean rule: place cat only if NOT safe AND NOT already a cat.
safe = {(first_row, first_col)} # first click is safe safe += get_neighbours(first_row, first_col) # so are neighbours while placed < 10: r, c = random.randint(0,9), random.randint(0,9) if (r,c) not in safe and NOT board[r][c].is_cat: board[r][c].is_cat = True placed += 1
get_neighbours(row, col) — finds surrounding squares
Returns up to 8 (row, col) pairs for the squares surrounding a given square. Edge/corner squares return fewer because some positions fall outside the board. Used by many other functions.
for dr in [-1, 0, 1]: # row offset: above, same, below for dc in [-1, 0, 1]: # col offset: left, same, right if dr==0 and dc==0: continue # skip the square itself if 0 <= row+dr < 10 and 0 <= col+dc < 10: neighbours.append((row+dr, col+dc))
reveal_cell(row, col) — the most important function
Called every time you left-click. Applies boolean checks, handles first click, cat hits, flood fill, and win check. This is the heart of the game.
if cell.is_revealed OR cell.is_collared: return # Boolean guard if first_click: place_cats(row, col) # safe zone around this click start_time = time.time() cell.is_revealed = True if cell.is_cat: # hit a cat! game_over = True; show_all_cats() elif cell.neighbour_count == 0: flood_fill(row, col) # auto-open neighbours check_win()
flood_fill(start_row, start_col) — BFS algorithm
When you open an empty square (neighbour_count=0), this function auto-opens all connected empty squares using a queue. It keeps adding unvisited empty neighbours to the queue until the whole empty region is revealed.
queue = deque([(start_row, start_col)]) while queue: row, col = queue.popleft() # take next square for nr, nc in get_neighbours(row, col): # Boolean: only expand if NOT revealed AND NOT collared AND NOT cat if not revealed and not collared and not cat: cell.is_revealed = True if cell.neighbour_count == 0: queue.append((nr, nc)) # keep going!
check_win() — called after every reveal
Counts all cells where is_cat=False AND is_revealed=True. If that count equals total safe squares (90), every safe cell is open — you win!
total_safe = 100 - 10 # = 90 safe squares revealed_safe = count where NOT is_cat AND is_revealed if revealed_safe == total_safe: game_over = True # board cleared — show win message
toggle_collar(row, col) — right-click handler
Flips is_collared between True/False. Boolean rule: only works if NOT revealed. Updates the cat counter label on screen.
if cell.is_revealed: return # can't collar an open square cell.is_collared = NOT cell.is_collared # flip True↔False cat_counter = TOTAL_CATS - count_of_collars
update_timer() — self-scheduling, never freezes
Instead of a while loop (which would freeze the window), this function schedules itself to run again after 1000ms using window.after(). Boolean: only ticks if NOT game_over AND start_time is not None.
if NOT game_over AND start_time is not None: elapsed = time.time() - start_time timer_var.set(f"{elapsed}s") if NOT game_over: window.after(1000, update_timer) # schedule next tick

All 7 boolean rules used in this game

🖱️
accept_click = NOT game_over
Before doing anything with a click, check game hasn't ended. If game_over is True, ignore all clicks.
👁️
can_reveal = NOT is_revealed AND NOT is_collared
A cell can only be revealed if it hasn't already been opened AND it hasn't been collared. Both conditions must be false.
can_collar = NOT is_revealed
You can only place or remove a collar if the cell hasn't been revealed yet. Open cells can't be collared.
🐱
can_place_cat = NOT in_safe_zone AND NOT already_cat
When placing 10 cats: skip if the position is in the first-click's safe zone, AND skip if a cat is already there.
🔢
count_for_win = NOT is_cat AND is_revealed
A cell counts toward the win condition only if it's safe (not a cat) AND it's been opened. Used to check all 90 safe cells.
🌊
expand_bfs = NOT is_revealed AND NOT is_collared AND NOT is_cat
In flood fill: add a neighbour to the queue only if it hasn't been revealed, isn't collared, and isn't a cat. Three conditions, all with NOT.
⏱️
tick_timer = NOT game_over AND start_time is not None
The timer only counts while the game is active AND after the first click (start_time is only set on first click).

The auto-reveal algorithm

Flood fill — how empty areas auto-open

When you click a blank square (zero nearby cats), the game auto-reveals an entire connected region. This uses a computer science algorithm called Breadth First Search (BFS). Step through it to see how it works.

💡BFS in plain English: Start with one square. Check all its neighbours. If a neighbour is safe and empty, add it to a queue. Process the queue one by one, checking each square's neighbours. Keep going until the queue is empty.
Click "Step" to begin. The highlighted cell (green) was just clicked — it has 0 nearby cats.
Origin (clicked) In queue (to be processed) Visited (revealed) Wall (has a cat / number)
flood_fill() — the BFS code
def flood_fill(self, start_row, start_col):
    """BFS: auto-reveals connected empty region."""
    queue = deque([(start_row, start_col)])   # start with clicked cell

    while queue:                              # keep going until queue empty
        row, col = queue.popleft()            # take next cell from front
        for nr, nc in self.get_neighbours(row, col):
            cell = self.board[nr][nc]
            # Boolean: only if NOT revealed AND NOT collared AND NOT cat
            if not cell.is_revealed and not cell.is_collared and not cell.is_cat:
                cell.is_revealed = True
                cell.button.config(text="", bg=CELL_REVEAL, relief="sunken")
                if cell.neighbour_count == 0:
                    queue.append((nr, nc))    # empty: keep expanding
                # else: number cell — reveal it but don't expand further
Key terms

Glossary

Quick definitions for every programming concept used in this game.

class
A template/blueprint for creating objects. Like a cookie cutter — the class is the cutter, each object made from it is a cookie.
instance
One specific object made from a class. There are 100 Cell instances (one per square) and 1 CatstropheGame instance.
__init__()
The constructor — runs automatically when you create an instance. Sets up all the starting values. Called once per object.
self
Inside a class method, "self" refers to the current instance. self.is_cat means "this cell's is_cat variable."
boolean
A value that is either True or False. Named after mathematician George Boole. All the cell states (is_cat, is_revealed, is_collared) are booleans.
deque
A "double-ended queue" from Python's collections module. Used in BFS because adding/removing from both ends is O(1) — faster than a regular list.
BFS
Breadth First Search — explores all neighbours level by level. Used in flood fill to auto-reveal connected empty squares in the right order.
StringVar
A special tkinter variable. When you change its value, any label or widget connected to it updates automatically on screen. Used for the timer and cat counter.
lambda
A tiny anonymous function written inline. Used in _build_board() so each cell's click handler remembers its own (row, col): lambda r=row, c=col: ...
window.after()
Tkinter's way to schedule something to run later without freezing. Used by update_timer() to tick every 1000ms without blocking the UI.
O(n)
Big-O notation — how an algorithm scales. O(rows×cols) means the time grows proportionally with the board size. BFS is O(rows×cols) worst case.
safe zone
The first-clicked cell and its 8 neighbours. Cats are never placed here, guaranteeing the very first click is always safe and opens a good area.
